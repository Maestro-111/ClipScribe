# ClipScribe Frontend

A Vite + React + TypeScript single-page app for the ClipScribe web dashboard.
This now covers the jobs list, new-job form, SSE live job page, run inspector,
and post-run advisory chat described in `docs/web-app-plan.md` §10.

## Stack (and why each piece exists)

| Tool | Role |
|---|---|
| **Vite** | Dev server (HMR) + production bundler. Also runs the `/api` dev proxy. |
| **React 19 + TypeScript (strict)** | UI, fully typed. |
| **TanStack Router** | File-based, type-safe routing. Files in `src/routes/` → URLs. |
| **TanStack Query** | Server-state cache: fetching, caching, refetch/polling, loading+error state. |
| **Zustand** | Available for future shared client state; the live-job page currently uses a local reducer. |
| **Tailwind v4** | Utility CSS via `@tailwindcss/vite`. No config file — theme lives in `src/styles.css`. |
| **openapi-typescript** | Generates `src/api/types.ts` from the API's OpenAPI schema. |
| **openapi-fetch** | Tiny typed HTTP client built on those types. |

> shadcn/ui (in the plan) is a component *generator*, not a runtime dep. Add it
> later with `pnpm dlx shadcn@latest init` — it vendors component source into the
> repo. Deferred to keep the bootstrap minimal.

## First run

```bash
# 1. Start the backend API (from backend/, another terminal).
#    Normal dashboard browsing needs the builder/DB dependencies loaded.
cd ../backend
uv run uvicorn app.main:app --reload

# 2. Install deps and generate the typed API client (needs the API running).
cd ../frontend
pnpm install
pnpm gen:api           # writes src/api/types.ts from http://localhost:8000/openapi.json

# 3. Start the dev server.
pnpm dev               # http://localhost:5173
```

`pnpm dev` generates `src/routeTree.gen.ts` on start (gitignored). If your editor
shows a red squiggle importing it in `main.tsx`, run `pnpm dev` once.

## How the pieces connect (the mental model)

```
Browser (:5173)
  └─ fetch("/api/jobs")            openapi-fetch, typed from src/api/types.ts
        └─ Vite proxy rewrites /api → :8000   (vite.config.ts; avoids CORS)
              └─ FastAPI /jobs      (backend/app/routes/jobs.py)
```

The live job page also opens `EventSource("/api/jobs/{id}/events")`, which
replays and tails Redis Stream progress events through the same Vite proxy.

- **Routing:** a file like `src/routes/runs.$runId.tsx` becomes `/runs/:runId`.
  The `$` marks a dynamic segment; `.` is a path separator. The Vite plugin
  regenerates `routeTree.gen.ts` whenever you add/rename a route file.
- **Type-safety across the stack:** `pnpm gen:api` turns the Python Pydantic
  models into a TS `paths` type. `openapi-fetch` is generic over it, so typed
  API calls reject wrong endpoints or malformed request bodies at compile time.
  Re-run `gen:api` after any backend API change (this is the anti-drift step,
  plan §9.12). Multipart uploads and a few lifecycle calls use plain `fetch`
  where it is simpler than wrapping them in `openapi-fetch`.
- **Data fetching:** components never call `fetch` directly. They use the hooks
  in `src/api/hooks.ts` (`useJobs`, `useJob`, `useCreateJob`, …). Reads are
  `useQuery` (cached); writes are `useMutation` (invalidate the cache on success).

## Layout

```
src/
  main.tsx              App entry: mounts React, QueryClient, RouterProvider.
  styles.css            Tailwind import.
  routeTree.gen.ts      GENERATED by the router plugin (gitignored).
  api/
    types.ts            GENERATED by openapi-typescript (`pnpm gen:api`).
    client.ts           openapi-fetch client + error unwrapping.
    hooks.ts            TanStack Query hooks over the client.
  components/
    ChatPanel.tsx       Post-run advisory chat, streamed over SSE.
  lib/format.ts         Display helpers.
  routes/
    __root.tsx          App shell (nav + <Outlet/>).
    index.tsx           "/"            Jobs list.
    jobs.$jobId.tsx     "/jobs/:jobId" Live job progress.
    jobs.new.tsx        "/jobs/new"    New-job form.
    runs.$runId.tsx     "/runs/:runId" Run inspector.
```

## Current UI scope

- Jobs list: status filter, polling, running-job progress bars, inspect links for completed runs, queued-job stop, failed/canceled retry, and terminal-job delete.
- New job: create `full` or `extract` jobs from an existing input video or upload; parser-only jobs are disabled in the form for now.
- Live job page: SSE-driven progress bar, phase tree, current-shot panel, and log tail from `GET /jobs/{job_id}/events`.
- Run inspector: raw video playback with SVG boxes for tracked objects and OCR text, shot/audio timeline, parser criteria table, tracked video download, and advisory chat.
- Not yet built: running-job cooperative cancellation, the fuller inspector layer controls from the plan, and failed-criterion "ask about this" shortcuts.

## Scripts

- `pnpm dev` — dev server with HMR.
- `pnpm build` — typecheck + production build to `dist/`.
- `pnpm typecheck` — types only (no build).
- `pnpm gen:api` — regenerate `src/api/types.ts` from the running API.
